运营之光2.0：我的互联网运营方法论与自白

更新于2022-11-07|Learn
|字数总计:6.2k|阅读时长:17分钟|阅读量:

在互联网行业内，“ 运营 ” 这个职能发展到一定阶段后，往往更需要有成熟的知识体系和工作方法来给予行业从业者以指引。

本书尤其难得之处在于：它既对 “ 什么是运营 ” 这样的概念认知类问题进行了解读，又带有大量实际的工作技巧、工作思维和工作方法，还包含了很多对于运营的思考、宏观分析和建议，可谓内容完整而全面，同时书中加入了作者亲历的大量真实案例，让全书读起来深入浅出、耐人寻味。

从内容的受众来说，它既有面向初入互联网行业的运营从业者的具体工作方法的讲解和建议，又有适合 3～5 年运营从业者阅读的一些案例解析、思考方法分享，也有适合创业者、互联网公司高管阅读的对一些运营体系搭建、不同类型产品所适合的运营方法等更为宏观的问题的解读。

运营是什么

1、互联网公司的运营岗位和工作职责高度不标准，产品负责界定和提供长期用户价值，运营负责创造短期用户价值+协助产品完善长期价值。运营作为产品和用户的纽带和桥梁，帮助产品与用户更好地建立关系，以及更好地维系这种关系。

2、经典意义上的四大运营模块如下。

运营工作
内容运营 围绕内容的生产和消费搭建良性循环。
用户运营 围绕用户的新增—留存—活跃—传播以及用户之间的价值供给关系建立良性循环。
活动运营 围绕活动策划、资源确认、宣传推广、效果评估等一系列流程做好全流程项目推进、进度管理和执行落地。
产品运营 通过各种手段可拉升某个产品的特定数据。

3、其他运营岗位包括新媒体运营、APP 商店推广运营、SEO／SEM 运营、广告投放运营／流量运营、淘宝店铺运营等。

4、不同业务类型的产品和公司有不同的运营工作内容，但都可能会通过新媒体等途径可建设品牌和维护用户。

运营重心
工具类产品 注重效率和体验，通常产品大于运营，在较长的时间里，运营端最重要的关注点是用户增长。
社交或社区类产品 注重社交氛围、话题和玩法，通常产品和运营并重。
内容类产品 运营核心是持续做出独特、高质量的好内容，并包装这些内容，让用户更易于消费。
电商类产品 包括商品和品类运营、促销活动的策划和落地执行、推广和流量建设、用户关怀和用户维系、基础建设。
平台类产品 注重节奏、策略和用户维护、品牌。
游戏类产品 运营重点是推广和收入。

5、传统产品和互联网产品的差异在于传统产品 = 功能 + 体验；互联网产品 = 功能 + 体验 + 用户参与价值。

6、用户参与价值包括用户的使用时间和关注给产品带来的额外价值；用户在互相给对方创造和提供价值；用户源于对产品的认可，参与产品改进。

7、运营的具体工作包括拉新 + 引流 + 转化和用户维系，要从宏观视角理解运营，且带着微观和落地的心态做好运营。

8、互联网公司运营工作流程是制定策略 → 分解指标 + 规划工作 → 执行落地 + 达成目标 → 检测数据 + 调整方向。

9、运营与市场的区别在于市场更关注品牌传播、塑造、定位、产品的无形价值；而运营更关注用户转化、使用、行为的促进和引导。

10、运营与产品的关系在于产品方向和产品形态决定运营思路；运营根据用户反馈和运营需求决定产品的迭代。

11、回溯整个互联网的 20 年，可以发现运营要通过较多样的手段和技能可更好地实现用户获取、用户付费和维系已有用户。

运营人要做什么

底层工作法

1、目标导向意识是指繁杂的运营工作有目标导向类的工作，如通过分类、归纳分析客服信息回复可给产品做改善建议。效率意识是指效率贯穿整个运营职业生涯，通过思考、判断和执行可找到投入产出较优的路径和方法，达成想要的效果。

2、运营的竞争力和信仰是指回报后置式的逻辑，更加强调专注给用户创造价值，且相信当创造的价值足够多时，用户一定愿意给予无条件的认可和回报。

3、精益运营是指当一个具体项目面临 N 多不确定因素时，往往存在一个最重要的因素，它可能是整个事件顺畅发生的核心前提；面对特别不确定的事情或假设，运营要用最低的成本搭建一个真实的用户应用场景，并验证用户在这个场景是否真的会产生所预期的行为。

4、优秀运营一定要做好两件事，一是爱上做局，即一个可让 N 多人一起参与且收获价值，在 N 多角色间实现价值匹配的氛围和状态；二是面对任何一个局面，一定要先找到破局点，并通过一系列动作使之成立和实现。

5、运营的三个底层工作方法包括拥有对新鲜事物的高度敏感；拥有对用户的洞察；学会更具打动力和说服力的表达。

6、关键性 “ 运营思维 ”的包括流程化思维、精细化思维、杠杆化思维、生态化思维。

7、、运营理解了产品经理的科学工作方法和流程后，甚至部分掌握了一些产品经理的思考方法和逻辑后，可能会降低无脑吐槽或者提出无脑需求的比例。

8、作为一名运营，如果能够时刻拥有一种 “ 寻找合理高效的产品机制来为运营服务 ” 的思考，这会带来更大的可能和便利。

9、懂得某些产品的逻辑、架构，甚至能够粗略对于实现成本有一些评估之后，会有助于大大降低与产品经理和研发之间的沟通成本。

10、深刻理解 “ 产品 ” 工作的本质，包括 MVP、精益、敏捷、“ 少即是多 ” 等产品理念和工作方法之后，会有助于用最合理的方法去推进很多工作的开展，能够在整个业务链条中发挥更重要的作用，以及某些时候在与产品的话语权争夺中占据主导地位。

核心能力

1、靠谱的运营会把一个大目标的所有工作任务都拆分到极细、极具体，尽力让 70% 以上的事情变得对自己可知可控，只留下 30% 的不确定性。

2、数据贯穿整个运营生涯，其对运营的价值如下：数据可客观反映一款产品当前的状态好坏和所处阶段。若做完一件事但效果不好，数据可发现问题出在哪里。若想实现某个目标，数据可找到达成目标的最佳路径。极度精细的数据分析通过层层拆分可更了解用户，且对整个站内的生态更有掌控力。数据可能隐藏一些潜在的能让事情变得更好的线索和彩蛋。

3、内容可能是家底儿式的东西，小到文章标题，大到一篇小说，都是内容运营的范畴。从宏观来看，内容运营是持续关注内容从生产到消费再到流通和传播的全过程；从微观落地执行来看，内容运营要优先解决定位和调性。

4、做内容要关注长、短两条线。短线是尽一切努力促进内容被消费。长线是以一系列长期、持续的内容为载体，面向用户建立识别度和信任感，这需要建立定位和调性，其重要性在于一旦调性被成功树立，就可在用户心目中牢牢占据一个位置，进而大大降低以后建立用户认知的成本。

5、内容定位和调性最重要的两方面，一是必须要给自己的内容调性找到显著的不同和差异，找差异的逻辑跟品牌的定位高度相似，即品牌必须拥有一些与众不同的东西让用户记住、识别；二是即使调性真的找准，找到具体的发力点也不容易，想要把调性落到实处，最好先从内容背后提炼出一些棱角分明的标签，再用一系列具体、切实的行为动作支撑这些标签。

6、判断内容的好坏遵循三个基本原则：①好的内容有自己的主线，框架清晰的内容更易于用户的消费和理解。②逻辑较复杂或需要传递某种特别感觉的部分尽量用图表或图文的方式表现。③好的内容围绕用户的感知进行表达和叙述。

7、互联网行业有 PGC（专业内容生产模式）和 UGC（用户生产内容模式）两种内容生产模式。

生产环节
UGC 内容的初始化 → 少量用户加入生产 → 内容生产者激励 → 更多新用户进入 → 鼓励和引导更多用户加入。
PGC 选题策划 → 资料收集&整理 → 内容加工生产 → 内容组织和呈现。

8、内容的组织分为四个层次。

9、内容的流通包括对内流通和对外流通。

常见手法
对内流通 人为干预和组织、算法的智能推荐、依靠用户关系和用户行为（如点赞、评论）。
对外流通 通过产品机制、运营手段等鼓励用户自发把优质内容分享到第三方平台；运营人员主观挑选一部分优质内容分享到第三方平台以获取关注。

10、转化型文案的目的是引导用户完成某个特定行为，分为短文案和中长文案，其撰写的核心原则是只有先帮助用户建立认知，才有机会激发用户的兴趣。

11、短文案（如标题）提升转化率有以下两个原则。

要点
傍大款 有意识跟某些明显势能更高、影响力更强的人或事物形成关联。
颠覆认知 抛出某些可能会颠覆用户常识性认知，甚至有点不可思议的观点或言论，从而引发好奇心，撬动用户的点击访问意愿。

12、中长文案至少一两百字，内容信息量更大，需要逻辑和内容结构，逐步烘托和勾引用户的兴趣和欲望并最终形成转化，有以下三种写作方法。

写作方法
1 引起注意 → 激发兴趣 → 勾起欲望 → 促成行动。
2 代入情景 → 引起矛盾 → 提出问题 → 给出解决方案。
3 把用户在一个转化行为前可能会面临和思考的所有问题都依次列出来，然后一一对问题进行解答和说服用户。

13、内容本身的价值、打动力，与内容的传播、用户消费几率等，已经越来越合一了，用户在内容面前的话语权越来越大，而不再是渠道。一个内容能不能火，本质上越来越取决于用户是否愿意认可它、传播它，而不是核心渠道是否能够推荐它。

14、从宏观来看，用户运营是开源（拉新）、节流（减少流失）、维持（提高用户活跃和留存）、转付费；而从微观来看，任何一项业务发展到一定阶段都要更加精细化的管理和维护其用户，以便实现用户价值的最大化。

15、用户运营的工作职责分为两类，一是通过策略、机制等设计和优化对较大规模（如数十万量级）的用户整体批量运营，最终实现某个特定用户数据（通常是活跃用户数）的上升；二是运营较小规模（通常几十到几百人不等）的某一类特定用户，通过该类用户在产品生态中的贡献值，从而提高产品价值。

16、针对用户建立优质的成长路径，最常见的场景是解决新用户的留存问题，尤其是功能非常丰富的产品，在新用户初次使用的场景下，找到一条最有助于用户留存率提升的路径，往往意义重大。

17、针对现有用户分级，把运营变得更为精细化。先界定站内用户的关键属性和关键行为并提取对应的数据；再分析和比对数据，寻找用户属性和用户关键行为的相关性；一旦在找到用户属性和用户关键行为某种明确的相关性，就把某类用户单拎出来作为一类，针对性推送产品或服务。

18、针对用户设计面向用户行为的激励体系，用户激励体系是通过一系列的激励或约束导向的产品机制，更好鼓励或引导用户在产品站内发生特定行为。

19、通过分析流失用户 → 制定召回策略 → 根据策略确定召回内容三个步骤将沉默用户转化为活跃用户。

20、部分用户带动全体用户是指通过更有针对性和倾向性的运营高贡献值型用户，让他们持续活跃和创造价值，此后再通过他们的活跃或贡献的内容吸引或拉动更多其他用户活跃，常见于社交、社区、平台等产品。

21、某些 UGC 型产品可通过针对性地运营内容贡献型的用户，促使其能够持续生产优质内容，保证产品生态的完整；某些平台型产品分别面向供给端和消费端的部分用户，通过一些运营手段调动他们在站内生态中的积极性和提升其示范效应。

解决措施
1 明确重点抓取的用户，一般是有能力提供高质量内容和高质量服务的用户，且他们提供的服务往往更易受到用户喜欢和肯定.
2 明确这些用户在站内的行为标杆或示范。
3 明确能够给予用户的维系和激励手段：
① 人际关系维系：如通过大量日常沟通变成好朋友、举办定期线下活动等。
② 情感维系：如定期赠送暖心小礼品。
③ 价值提供：如招聘网站定期组织 HR 间的课程和分享会。
4 开始试运营，若跑得比较顺，则将其常态化。

22、某些产品刚上线可能会面临冷启动的问题（如社区类、平台类产品），此时可能要先找到一群种子用户，针对性地运营这一群人，以帮助产品建立起较好的冷启动条件，如令产品内部氛围更好、可消费内容更多、用户口碑更佳等，以更好地实现后续的发展和增长。

解决措施
1 定位产品适合的种子用户，并想办法找到他们，早期产品最理想的种子用户特征是容错性高且自带拉新传输属性。
2 维系这群用户并建立强关系。

23、产品用户增长的途径包括内容铺设、第三方渠道推广和广告投放、活动和事件等营销传播、PR 和品牌传播等。

24、撬动用户互动参与意愿的 8 个指导原则包括物质激励、概率性事件（如抽奖类活动）、营销稀缺感、激发竞争意识、赋予用户某种炫耀和猎奇的可能性、营造强烈情绪和认同感、赋予尊崇感和被重视感、通过对比营造超值感。

运营到底怎么做

商业逻辑

1、运营既要有具体的工作方法和技能，也要具备更高层面的思考判断能力，具有以下三个规律：带着短视的线性思维投入运营工作往往很难做好运营。一款产品在早期过于关注用户增长，甚至出现爆红等现象，往往反而会加速其死亡。早期产品的运营一定要围绕口碑进行。

2、根据不同的产品发展阶段或当前占据市场的份额的大小判定运营策略和运营规划如何制定，可把产品分为四种类型。

特点
探索期产品 上线时间不长，产品还在打磨，需要尚待验证，市场份额占据较小，此阶段运营目的是为能够服务好大量用户做好一切必要的准备，包括产品功能、产品使用体验、产品风格和氛围、服务能力等。
快速增长期产品 需求已得到验证，初步拥有一定市场份额，精品大量出现，此阶段核心目标是快速获取用户增长。
成熟稳定期产品 市场接近饱和，产品占据稳定的市场份额，增长空间很小，产品进入精细运营阶段，不同的产品模块有不同类型的用户，且有专人负责运营，给用户提供相应的服务和信息。
衰退期产品 意味着替代产品出现，用户开始批量流失，此阶段要注重老用户的维系和生命周期管理。

运营路径

1、根据不同的产品形态和业务类型判定运营策略和运营规划如何制定，可从三个维度评估：一是商业逻辑，二是典型用户行为频次，三是用户与其他用户间是否会通过产品而形成某种关系。

2、商业逻辑维度具有以下三个层面。

层面 核心
直接面向用户售卖某种商品或服务获利 能否找到足够好、足够多的商品，以足够低的成本、足够顺畅将其售卖出去，这要求具备商品和货源的选择和拓展能力、商品包装和营销能力、供应链全程服务能力等
免费 + 增值服务 是否能获取到足够多的用户，是否能让用户对产品形成依赖，以及是否可顺利撬动用户为增值服务买单，这要求具备免费试用用户的获取能力、用户的使用习惯和依赖性培养、用户日常使用行为到付费服务的路径搭建、最终的增值服务或第三方付费服务的售卖等
免费 + 流量 or 数据变现 是否能获取到足够多的用户，用户忠诚度是否足够，是否能持续带来新流量的数据或内容，是否愿意为之付费的内容、数据或特定氛围，这要求具备引导用户发生特定行为、搭建特定氛围的能力，持续维系住用户形成用户活跃度的能力，整合内容、数据和现有重点用户资源的能力

3、典型用户的行为频次维度具有以下三种类型。

类型 关注要点
用户一次性使用 产品更关注获客渠道的铺设和广告投放、销售转化的有效性、客单价，如培训课程（如托福雅思）、婚庆服务、殡葬服务等收费产品
用户中低频次使用 产品更关注渠道铺设、品牌传播，如汽车保养类产品、求职类产品等
用户高频次使用 产品更注重口碑营销和病毒营销，如阅读类产品、社交类产品等

4、用户间是否通过产品结成某种关系维度是指此类产品更注重氛围的打造，发展较慢且需要花费很大的精力逐渐建立和完善一些规则、边界和约束条件。

运营体系

1、基于产品逻辑搭建一个靠谱、稳定的运营体系，需要以下几个维度，一是保证基础业务的顺畅运转；二是尽量把产品的开源和节流变成固定动作；三是确保最关键用户行为的发生几率；四是界定核心用户和建立维系机制；五是阶段性通过活动、事件、营销等实现用户增长。👉 示例如下：

为了确保懂球帝 APP 的顺畅运转，运营端的基础工作可能包括保质保量地按时上传发布各类内容资讯、更新各类数据、榜单等；做好用户 UGC 内容的审核、筛选、组织和处理；按时发布各类足球装备并做好营销、客服等方面的相关工作，确保用户可在线顺畅地完成购买。懂球帝APP在没有额外重大事件发生的前提下，其稳定的用户增长可能来源于优质内容的用户自发对外分享、传播；优质内容的有意识对外传播；搜索引擎、应用商店等入口；每次发生足球类新闻的报道速度和解读角度与众不同。

因此，为了让懂球帝的开源变得相对稳定和持续，应该先围绕以上几点确保常规性运营工作的执行，包括结合用户场景，设计合理的分享流程，优化文案；可能需要专门有一个人负责优质内容的外推，包括寻找优质渠道、建立合作关系、常规性的内容外发和外部渠道维护等；做好 SEO、应用商店 ASO 等工作，并不断优化；组建专门的新闻资讯更新报道团队，建立报道机制；定期策划深度解读类内容，确保资讯报道速度和解读角度的独特性。

2、一个社区或社群从无到有到成熟的成长路径有几个关键节点，包括创建和初始化（找到主题）、确立信任感与价值、社区的去中心化（培养和发掘追随者 + 引导用户与用户建立联系）、社区的自成长（如何维持社区的成熟和活跃状态+如何通过话题的延展扩大自己的潜在用户数量）。

3、2B 和 2C 产品运营的本质区别在于 2B 产品面向的用户是 N 多个决策对象共存的决策链条，其运营更偏重于线上的传播或品牌、线下的活动和销售转化。

4、影响一家企业最终愿意购买和持续使用 2B 类产品的核心要点包括企业是否认可和接受企业自身存在某些问题且产品对解决这些问题有帮助；能否击破企业购买决策链当中的关键环节；用户在使用过程中能否真实感受到价值和效率的提升；用户是否对品牌有更强的信任度，从而在整个决策链流程中变得更加顺畅。
